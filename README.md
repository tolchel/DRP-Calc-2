# DRP Calculator — Калькулятор времени восстановления

Инструмент для сейлз-демонстраций, который наглядно показывает преимущество **Кибербакап** перед конкурентами по скорости восстановления ИТ-инфраструктуры.

Продавец вводит параметры заказчика (типы и объёмы активов, инфраструктуру резервного копирования) — приложение запускает Monte Carlo симуляцию и отображает сравнение двух сценариев в виде графиков плотности распределения.

## Как работает приложение

### Шаг 1 — Активы

Пользователь описывает инфраструктуру заказчика: типы объектов восстановления и объёмы данных.

- 4 типа активов: **Базы данных**, **Серверы**, **Файловые хранилища**, **Рабочие станции**
- Для каждого типа: количество активов + средний объём (ГБ)
- Автоматический расчёт подитогов и общего объёма
- Кнопка «Далее» заблокирована, пока суммарный объём = 0

### Шаг 2 — Инфраструктура

Пользователь задаёт параметры систем резервного копирования.

- **Ленточные библиотеки** — количество и пропускная способность драйвов (первая нельзя удалить, можно добавлять дополнительные)
- **СХД** — макс. скорость, количество потоков, скорость потока
- **Сеть** — скоростная (лента ↔ СХД) и ЛВС (СХД ↔ объекты)
- **Количество IT-инженеров**
- **Поправка на неопределённость** (%, влияет на время передачи данных)
- **Количество итераций Monte Carlo** (10 000–200 000, по умолчанию 10 000)
- Все ошибки валидации — на русском языке

### Шаг 3 — Результаты

Параллельно рассчитываются два сценария:

| Кибербакап | Конкурент |
|-----------|-----------|
| Прямая передача: лента → объекты | Двухфазная передача: лента → СХД → объекты |
| Скорость ограничена скоростной сетью | Фаза 1 — быстрая сеть, фаза 2 — ЛВС |
| 10 параллельных потоков | 10 потоков с СХД |

Для каждого сценария отображаются:
- **Метрические карточки:** Лучший / Худший / Вероятность лучшего / Среднее значение
- **KDE-диаграмма** плотности распределения времени восстановления (recharts)
- Вертикальные линии лучшего/худшего сценария + hover-тултипы
- Таблица разбивки по типам активов

### Ключевые технические решения

**Единственный HTML-файл.** `vite-plugin-singlefile` встраивает JS, CSS и Web Worker inline в `index.html`. Файл открывается через `file://` на Windows без localhost-сервера — это ключевое требование для сейлз-демонстраций.

**Web Worker для симуляции.** Monte Carlo (до 200 000 триалов) выполняется в отдельном потоке через `?worker&inline` import. Прогресс передаётся ~20 сообщениями — UI не зависает.

**Честное сравнение сценариев.** Оба сценария (Кибербакап и Конкурент) вычисляются в одном цикле с одними и теми же случайными величинами (`networkFactor`, `engineerFactor`) на каждый триал. Разница в результатах обусловлена только логикой сценариев, а не шумом.

**KDE по алгоритму Сильвермана.** Гауссово ядро + правило Сильвермана для подбора bandwidth. 200 000 триалов → 300 точек KDE. Fallback h=1.0 при нулевой дисперсии.

**Единицы измерения.** Внутри движка все расчёты в **часах**. `STARTUP_MINUTES=30` конвертируется при расчёте. Поправка на неопределённость применяется только к времени передачи данных, не к времени запуска.

**Модель инженеров.** `verifiersPerShift = floor(N / 3)`, где N — количество инженеров. При N < 3 смены без верификатора добавляют паузу 480 мин (8 ч) на объект.

## Модель симуляции

### Порядок восстановления

Активы восстанавливаются последовательно по приоритету: **БД → Серверы → Файловые хранилища → Рабочие станции**

### Случайные факторы (на каждый триал)

| Фактор | Распределение | Применяется к |
|--------|--------------|---------------|
| `networkFactor` | Uniform(0.8, 1.2) | Скорость сети (общий для обоих сценариев) |
| `engineerFactor` | Uniform(0.5, 1.0) | Доступность IT-инженеров |
| `uncertaintyFactor` | 1 + Uniform(0, uncertaintyPct) | Время передачи данных на актив |

### Формулы

**Кибербакап** (прямая передача лента → объекты):
```
throughput = min(∑tapeSpeed, fastNetworkCap) × networkFactor
transferTime = volumeGB / throughput
```

**Конкурент** (двухфазная передача):
```
phase1 = volumeGB / min(∑tapeSpeed, fastNetworkCap)   # лента → СХД
phase2 = volumeGB / min(sanSpeed, lanCap)              # СХД → объекты
totalTransfer = phase1 + phase2
```
