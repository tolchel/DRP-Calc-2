    Create src/hooks/useSimulation.ts — a React hook that manages the Web Worker lifecycle for running simulations.

    The hook signature:
    ```typescript
    function useSimulation(): {
      run: (form: WizardFormData) => void
      cancel: () => void
      progress: number | null   // null = idle, 0-100 = running/complete
      result: SimulationResult | null
      isRunning: boolean
    }
    ```

    Implementation requirements:
    - Use useRef<InstanceType<typeof SimWorker> | null> to hold the worker instance (matches existing App.tsx pattern)
    - Import SimWorker from '../workers/simulation.worker?worker&inline' (CRITICAL: this exact syntax — do NOT use new Worker(new URL(...)))
    - Import wizardToSimInput from '../types/wizard' to convert form data before postMessage
    - run(form): terminate any existing worker, create new SimWorker, set progress=0, result=null, isRunning=true, postMessage({ type: 'run', data: wizardToSimInput(form) })
    - onmessage handler: 'progress' → setProgress(e.data.percent); 'result' → setResult(e.data.data), setProgress(100), setIsRunning(false), worker.terminate()
    - cancel(): terminate worker, reset progress=null, isRunning=false (leave result as-is)
    - Clean up on unmount: useEffect return → workerRef.current?.terminate()
    - Import WorkerInMessage, WorkerOutMessage from '../types/worker'
    - Import SimulationResult from '../types/simulation'

    Use useState for progress, result, isRunning. Use useRef for workerRef.
    Export as named export: export function useSimulation()
