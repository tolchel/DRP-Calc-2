  <behavior>
**computeKDE(samples: number[], maxPoints = 250): KDEPoint[]**

1. Sort samples and compute min/max for x-axis range
2. Compute Silverman bandwidth:
   ```
   n = samples.length
   mean = sum(samples) / n
   variance = sum((x - mean)^2) / (n - 1)
   stdDev = sqrt(variance)
   sorted = samples.sort()
   iqr = sorted[floor(0.75 * n)] - sorted[floor(0.25 * n)]
   s = min(stdDev, iqr / 1.34)
   h = 1.06 * s * n^(-0.2)
   ```
3. Generate maxPoints equidistant x values from min to max
4. For each x, compute Gaussian kernel density:
   ```
   density[i] = (1 / (n * h)) * sum(gaussianKernel((x[i] - sample) / h) for sample in samples)
   gaussianKernel(u) = (1 / sqrt(2π)) * exp(-0.5 * u^2)
   ```
5. Return array of {x, density} pairs

**Edge cases:**
- If samples.length < 2, return [{x: samples[0], density: 1}] or throw — document choice
- If all samples are identical (zero variance), h = 1.0 (fallback to avoid divide-by-zero)
- maxPoints is capped at 300 regardless of argument (enforce the SIM contract)

**Performance:**
- For 200,000 samples and 250 output points: 200,000 × 250 = 50M kernel evaluations
- This is called once after all trials complete (not per-trial) — acceptable in the Worker thread
- If too slow: subsample to 10,000 samples for bandwidth calculation, use full set for density
  </behavior>
